🐚 Shell 脚本知识整理手册

一、变量与环境

1. 变量操作

1set -- 'a' 'b'
  • 清空所有位置参数，并重新设置 $1 $2 ...
1unset 变量名
  • 删除变量（局部变量 + 环境变量）
1export 变量名=值
2declare -x 变量名=值
  • 导出为环境变量（子进程可用）
1env
  • 查看当前环境变量
1declare
  • 声明变量（可用于定义类型/属性）

2. 特殊变量

变量 含义
$0 脚本名称
$1 ~ $9 参数1~9
${10} 第10个参数
$# 参数个数
$* 所有参数（整体）
$@ 所有参数（逐个）
$? 上一个命令执行状态（0成功）
$$ 当前进程PID
$! 上一个后台进程PID
$_ 上一个命令的最后一个参数

3. 路径处理

1dirname 路径   # 获取目录
2basename 路径  # 获取文件名

二、参数展开（变量高级用法）

1. 基本用法

1${arg}          # 获取变量值
2${#arg}         # 长度
3${arg:offset}   # 截取
4${arg:offset:len}

2. 字符串处理

1${arg#word}     # 从前删除最短匹配
2${arg##word}    # 从前删除最长匹配
3${arg%word}     # 从后删除最短匹配
4${arg%%word}    # 从后删除最长匹配

3. 替换

1${arg/pattern/string}    # 替换一次
2${arg//pattern/string}   # 全部替换

4. 默认值机制（重点⭐）

1${arg:-word}   # 为空使用默认值
2${arg:=word}   # 为空赋值
3${arg:?word}   # 为空报错
4${arg:+word}   # 不为空才使用

三、算术运算

1((a=1+2))
2let a=1+2
3expr 1 + 2
4$[1+2]
5echo "1+2" | bc
6awk 'BEGIN{print 1+2}'

推荐：(( )) 最常用 👍

4. 数值比较

test / [] [[]] / (())

-gt >

🐚 Shell 进阶实战手册（企业级版）

一、企业级脚本模板（强烈推荐直接复用⭐）

 1#!/usr/bin/env bash
 2
 3#######################################
 4# 脚本名称：demo.sh
 5# 描述：示例脚本模板
 6# 作者：your_name
 7# 时间：$(date +%F)
 8#######################################
 9
10# ===== 严格模式（必须开启）=====
11set -euo pipefail
12
13# ===== 全局变量 =====
14SCRIPT_NAME=$(basename "$0")
15LOG_FILE="/tmp/${SCRIPT_NAME}.log"
16
17# ===== 日志函数 =====
18log() {
19    local level="$1"
20    local msg="$2"
21    echo "$(date '+%F %T') [$level] $msg" | tee -a "$LOG_FILE"
22}
23
24# ===== 错误处理 =====
25error_exit() {
26    log "ERROR" "$1"
27    exit 1
28}
29
30# ===== 捕获异常 =====
31trap 'error_exit "脚本异常退出，行号：$LINENO"' ERR
32
33# ===== 清理函数 =====
34cleanup() {
35    log "INFO" "执行清理操作..."
36}
37trap cleanup EXIT
38
39# ===== 主逻辑 =====
40main() {
41    log "INFO" "脚本开始执行"
42
43    # 示例
44    echo "Hello Shell"
45
46    log "INFO" "脚本执行完成"
47}
48
49main "$@"

二、严格模式（生产必备🔥）

1set -euo pipefail
参数 作用
-e 任意命令失败立即退出
-u 使用未定义变量报错
-o pipefail 管道中任一失败都算失败

👉 没有这个，生产事故概率 +80%

三、日志规范（非常重要⭐）

1. 标准日志格式

12026-03-23 12:00:00 [INFO] 启动任务
22026-03-23 12:00:01 [ERROR] 数据库连接失败

2. 推荐封装

1log() {
2    echo "$(date '+%F %T') [$1] $2"
3}

3. 输出到文件 + 控制台

1echo "message" | tee -a logfile.log

四、错误处理体系（核心能力🔥）

1. 统一错误出口

1error_exit() {
2    echo "ERROR: $1"
3    exit 1
4}

2. 捕获错误

1trap 'echo "错误发生在第 $LINENO 行"; exit 1' ERR

3. 推荐写法

1cp file.txt /target || error_exit "复制失败"

五、参数处理（标准写法）

1. getopts（推荐⭐）

1while getopts "f:o:" opt; do
2    case $opt in
3        f) file="$OPTARG" ;;
4        o) output="$OPTARG" ;;
5        *) echo "Usage: $0 -f file -o output"; exit 1 ;;
6    esac
7done

2. 示例

1./script.sh -f input.txt -o out.txt

8# 汇总
