📦 主流容器编排系统汇总

1. Kubernetes（事实上的行业标准）

特点

  • 自动调度
  • 自愈能力
  • 服务发现
  • 滚动升级
  • 水平扩容

  • 社区最活跃

  • 扩展能力强

截至 2026 年，Kubernetes 已经成为事实上的容器编排标准。很多产品（OpenShift、Rancher、各大云厂商托管服务）本质上都是围绕 Kubernetes 构建，其它早期方案如 Docker Swarm、Mesos 已逐渐退出主流。对于新项目，优先学习和使用 Kubernetes 生态通常是最具长期价值的选择。 🚀

Kubernetes Master 节点和 Node 节点组件详解

在 Kubernetes 集群中，节点大致分为：

  • Control Plane（Master 节点）
  • Worker Node（Node 工作节点）

其中：

Master 节点负责「管理和调度」，Node 节点负责「运行容器」。

可以理解成：

 1                 Control Plane（Master）
 2          ┌───────────────────────────┐
 3          │ kube-apiserver            │
 4          │ etcd                      │
 5          │ kube-scheduler            │
 6          │ kube-controller-manager   │
 7          └─────────────┬─────────────┘
 8                        │
 9         ───────────────┼────────────────
10                        │
11           Worker Node 1       Worker Node 2
12      ┌─────────────────┐ ┌─────────────────┐
13      │ kubelet         │ │ kubelet         │
14      │ kube-proxy      │ │ kube-proxy      │
15      │ Container Runtime│ │ Container Runtime│
16      │ Pod             │ │ Pod             │
17      └─────────────────┘ └─────────────────┘

☸️ Kubernetes 核心资源汇总

Kubernetes 中的资源（Resource）是通过 API 对象来描述集群状态的。可以把它们理解成：

Kubernetes = 一组资源对象 + Controller 持续维护期望状态

按照功能，可以分为 工作负载资源、服务发现资源、配置资源、存储资源、调度资源、安全资源、元数据资源 等几大类。

一、工作负载资源（Workload）

这一类资源负责运行应用。

资源 Kind 作用
Pod Pod 最小调度单位
ReplicaSet ReplicaSet 保证 Pod 副本数
Deployment Deployment 无状态应用部署
StatefulSet StatefulSet 有状态应用部署
DaemonSet DaemonSet 每个节点运行一个 Pod
Job Job 一次性任务
CronJob CronJob 定时任务
ReplicationController ReplicationController 老版本副本控制器（已很少使用）

最小运行单元。

例如：

  • 一个或多个容器

    • Network
    • Volume
    • Namespace
